# Gomid 工程编译步骤（WSL / Ubuntu）

> 生成时间：2026-05-12 14:36:01（已修订）

---

## 1. 安装 Go 环境

**推荐方式：手动下载指定版本**（apt 源的 Go 版本通常过旧，不建议用于生产）

```bash
sudo apt install curl -y
curl -LO https://dl.google.com/go/go1.22.1.linux-amd64.tar.gz
sudo rm -rf /usr/local/go && sudo tar -C /usr/local -xzf go1.22.1.linux-amd64.tar.gz
```

写入 PATH 并立即生效：

```bash
echo 'export PATH=$PATH:/usr/local/go/bin:~/go/bin' >> ~/.bashrc
source ~/.bashrc
```

验证：

```bash
go version
# 期望输出：go version go1.22.1 linux/amd64
```

> ⚠️ WSL 用户：请将项目放在 Linux 文件系统（如 `~/workspace/`），不要放在 `/mnt/c/...`，否则 I/O 极慢。

---

## 2. 下载 gomid 工程并安装工具链

> 前置条件：需确保 SSH Key 已配置到 GitLab，且 WSL 可以访问内网地址 `10.110.80.44`。
> 可用 `ping 10.110.80.44` 提前验证连通性。

```bash
mkdir -p ~/workspace && cd ~/workspace
git clone git@10.110.80.44:platform/gomid.git
cd gomid
make setup
```

成功时输出如下：

```
Setup environment for linux(x86_64)
√ go version go1.22.1 linux/amd64
√ libprotoc 3.12.4
√ protoc-gen-go
√ protoc-gen-gogo
√ protoc-gen-swagger
√ mockery
√ goimports
√ nats-server: v2.6.5 ...
indepent packages are installed
```

若未全部通过，按照以下 1）、2）步骤补充操作后重试。

### 1）创建 proto include 路径

> 注意：下方第二行路径 `/home/<用户名>/go/...` 请替换为 `make setup` 输出中提示的实际路径，需要 sudo 执行。

```bash
mkdir -p /usr/local/include/google
# 将 <用户名> 替换为你的实际用户名，或用 $(go env GOPATH) 动态获取
sudo cp -r "$(go env GOPATH)/pkg/mod/github.com/grpc-ecosystem/grpc-gateway@v1.16.0/third_party/googleapis/google/api" \
    /usr/local/include/google/
```

### 2）配置私有 Go 包仓库

```bash
go env -w GONOSUMDB=git.joint520.com
go env -w GOPRIVATE=git.joint520.com   # ← 只填域名，不要拼接本地文件路径
go env -w GOPROXY=https://goproxy.cn,direct
```

---

## 3. 编译 gomid

如果 `make` 报错，先执行 `go mod tidy` 安装缺失依赖，再重试：

```bash
make
# 如果报错：
go mod tidy
make
```

---

## 4. 生成 go.work 多模块工作区

```bash
cd ~/workspace
go work init
go work use gomid
```

（后续生成 sample 工程后，再执行 `go work use sample`）

---

## 5. 生成并运行 Go 工程

```bash
cd ~/workspace
./gomid/3rd_party/gogen/gogen_linux_amd64 create --type http git.joint520.com/platform/demo gomid/testdata/proto/sample.proto
cd demo
go work use .   # 将 demo 添加到 go 工作区
make
make run
```

---

## 6. 交叉编译 ARM 格式（可选）

```bash
GOARCH=arm GOOS=linux go build -o ./bin/demo ./cmd/demo
```

---

## 7. 更新 proto 文件（可选）

在工程目录下，使用 gogen update 命令重新生成代码：

```bash
cd ~/workspace/demo
../gomid/3rd_party/gogen/gogen_linux_amd64 update --proto demo.proto
```
